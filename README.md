# 🔬 ArXiv Lens

Scrape, summarize, and browse CS papers from arXiv — all explained in plain English using Groq + LLaMA 3.

---

## Stack

| Layer | Tool |
|---|---|
| UI | Streamlit |
| Scraping | BeautifulSoup + Requests |
| Summarization | Groq API (LLaMA 3.3 70B) |
| Storage | Supabase (PostgreSQL) |

---

## Setup

### 1. Clone & install dependencies

```bash
git clone <your-repo>
cd arxiv_summarizer

python -m venv venv
source venv/bin/activate        # Windows: venv\Scripts\activate

pip install -r requirements.txt
```

### 2. Create your `.env` file

```bash
cp .env.example .env
```

Then open `.env` and fill in:

```
GROQ_API_KEY=your_groq_api_key_here
SUPABASE_URL=https://your-project-id.supabase.co
SUPABASE_KEY=your_supabase_anon_key_here
```

**Where to get keys:**
- **Groq API key** → [console.groq.com](https://console.groq.com) (free tier available)
- **Supabase URL & key** → Your Supabase project → Settings → API

---

### 3. Create the Supabase table

Go to your Supabase project → **SQL Editor** → run this:

```sql
CREATE TABLE papers (
    id          BIGSERIAL PRIMARY KEY,
    arxiv_id    TEXT UNIQUE NOT NULL,
    arxiv_url   TEXT NOT NULL,
    title       TEXT NOT NULL,
    authors     TEXT,
    category    TEXT,
    abstract    TEXT,
    summary     TEXT,
    scraped_at  TIMESTAMPTZ DEFAULT NOW()
);

-- Index for faster category filtering
CREATE INDEX idx_papers_category ON papers(category);

-- Index for faster date sorting
CREATE INDEX idx_papers_scraped_at ON papers(scraped_at DESC);
```

---

### 4. Run the app

```bash
streamlit run app.py
```

Open [http://localhost:8501](http://localhost:8501) in your browser.

---

## How to Use

1. **Sidebar** — Select the CS sub-categories you want to scrape (AI, CV, NLP, ML, SE)
2. **Max papers slider** — Choose how many papers to process per category (5–50)
3. **Fetch & Summarize button** — Starts scraping arXiv, fetches abstracts, sends them to Groq for summarization, and stores results in Supabase
4. **Dashboard tab** — Browse all summarized papers with search and category filter
5. **Fetch Log tab** — Live progress during a fetch run

---

## Notes

- Papers already in the database are automatically skipped on re-runs (deduplication by arXiv ID)
- Summaries are generated by **LLaMA 3.3 70B** via Groq — fast, free-tier friendly, and better than Gemma for instruction-following
- arXiv's "recent" pages update daily — best to run once per day for fresh papers
- Groq free tier: 30 req/min — the app adds a 0.5s delay between papers to stay within limits
